# 项目工时管理系统 - 需求规格说明书

## 1. 认证模块

### 1.1 登录
- 用户输入：用户名 + 密码 + 图形验证码
- 验证码：后端生成图片（4位数字/字母），返回图片流 + 验证码ID
- 校验顺序：验证码 → 用户名密码 → 用户状态
- 成功：返回 JWT token，含 userId、username、role
- 失败：用户名或密码错误 / 验证码错误 / 账号已禁用，统一返回错误码

### 1.2 退出
- 前端清除 localStorage 中的 token，跳转登录页
- 服务端不做 token 吊销（JWT 无状态）

### 1.3 密码修改
- 需旧密码验证
- 旧密码正确 → 新密码 BCrypt 加密更新
- 新密码长度 ≥ 6 位

### 1.4 首次登录强制修改
- 新增用户默认密码：123456
- 首次登录时，服务端返回标记 `firstLogin: true`
- 前端检测到 `firstLogin` 后引导用户修改密码
- 修改成功后，首次登录标记清除

## 2. 用户管理

### 2.1 字段定义

| 字段 | 类型 | 约束 | 说明 |
|:-----|:-----|:-----|:-----|
| id | BIGINT | PK, 自增 | |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(255) | NOT NULL | BCrypt 加密 |
| name | VARCHAR(50) | NOT NULL | 姓名 |
| email | VARCHAR(100) | 可空 | 邮箱 |
| phone | VARCHAR(20) | 可空 | 手机号 |
| department | VARCHAR(50) | 默认'研发与交付中心' | 部门 |
| role | VARCHAR(20) | NOT NULL, 默认'USER' | USER / PM / DEPT_MANAGER / ADMIN |
| status | TINYINT | NOT NULL, 默认1 | 1启用 / 0禁用 |
| first_login | TINYINT | NOT NULL, 默认1 | 1首次登录 / 0已修改 |
| create_time | DATETIME | 默认 CURRENT_TIMESTAMP | |
| update_time | DATETIME | ON UPDATE CURRENT_TIMESTAMP | |
| is_deleted | TINYINT | 默认0 | 逻辑删除 |

### 2.2 功能
- 新增用户：管理员填写信息，密码自动设为 123456，first_login=1
- 编辑用户：可修改除密码外的所有字段
- 删除用户：逻辑删除（is_deleted=1）
- 重置密码：管理员可重置用户密码为 123456，first_login 置 1
- 角色分配：修改 role 字段
- 启用/禁用：修改 status 字段
- 用户列表：分页查询，支持用户名/姓名搜索

## 3. 项目管理

### 3.1 字段定义

| 字段 | 类型 | 约束 | 说明 |
|:-----|:-----|:-----|:-----|
| id | BIGINT | PK, 自增 | |
| name | VARCHAR(200) | NOT NULL | 项目名称 |
| description | TEXT | 可空 | 项目描述 |
| manager_id | BIGINT | NOT NULL, FK→tb_user | 项目经理（仅限 PM 角色） |
| start_date | DATE | 可空 | 开始日期 |
| end_date | DATE | 可空 | 结束日期 |
| status | VARCHAR(20) | 默认'ACTIVE' | ACTIVE(进行中) / FINISHED(已结束) |
| create_time/update_time/is_deleted | | 同用户表 | |

### 3.2 功能
- 新增项目：项目经理只能从 PM 角色用户中选择
- 编辑项目：可修改除 ID 外的所有字段
- 删除项目：逻辑删除
- 项目列表：分页查询，支持名称搜索
- 仅项目经理和管理员可管理项目

### 4.1 字段定义

| 字段 | 类型 | 约束 | 说明 |
|:-----|:-----|:-----|:-----|
| id | BIGINT | PK, 自增 | |
| project_id | BIGINT | NOT NULL, FK→tb_project | 所属项目 |
| name | VARCHAR(200) | NOT NULL | 模块名称 |
| description | TEXT | 可空 | 模块描述 |
| estimated_hours | INT | NOT NULL, 默认0 | 预估工时（预算上限，整数） |
| create_time/update_time/is_deleted | | 同用户表 | |

### 4.2 功能
- 新增/编辑/删除模块（逻辑删除），属于某项目
- 模块列表：按项目查询
- 仅项目经理可管理（所属项目的 PM 或管理员）

## 5. 工时填报

### 5.1 字段定义

| 字段 | 类型 | 约束 | 说明 |
|:-----|:-----|:-----|:-----|
| id | BIGINT | PK, 自增 | |
| user_id | BIGINT | NOT NULL, FK→tb_user | 填报人 |
| project_id | BIGINT | NOT NULL, FK→tb_project | 所属项目 |
| module_id | BIGINT | FK→tb_task_module, 可空 | 任务模块（PM可为空） |
| work_date | DATE | NOT NULL | 工作日期 |
| hours | DECIMAL(4,1) | NOT NULL | 工时（可任意填） |
| content | TEXT | 可空 | 工作内容 |
| status | VARCHAR(20) | NOT NULL, 默认'PENDING' | PENDING / APPROVED / REJECTED |
| create_time/update_time/is_deleted | | 同用户表 | |

### 5.2 功能
- 填报：选择日期+项目+模块+小时数+工作内容
- 同一日期可填多个项目
- 预算约束（普通用户）：按项目+模块累计工时 ≥ estimated_hours 时，该模块不可再报（提示"预算已达上限"）
- PM 填报：可不选模块（module_id 为空）
- 已审批工时：不可修改/删除
- 未审批工时：可修改/删除（重新提交后 status 变回 PENDING）
- 工时列表：按用户/项目/日期范围筛选

## 6. 工时审批

### 6.1 字段定义

| 字段 | 类型 | 约束 | 说明 |
|:-----|:-----|:-----|:-----|
| id | BIGINT | PK, 自增 | |
| work_hour_id | BIGINT | NOT NULL, FK→tb_work_hour | 关联工时记录 |
| approver_id | BIGINT | NOT NULL, FK→tb_user | 审批人 |
| status | VARCHAR(20) | NOT NULL | APPROVED / REJECTED |
| comment | TEXT | 可空 | 审批意见 |
| approve_time | DATETIME | 可空 | 审批时间 |
| create_time | DATETIME | 默认 CURRENT_TIMESTAMP | |

### 6.2 审批流程
- 普通用户报的工时 → PM 审批
- PM 报的工时 → 部门经理审批
- 部门经理不填报工时
- 支持批量审批（勾选多条，一次性通过/驳回）
- 驳回后用户可修改重新提交 → 新生成一条审批记录（原记录保留）
- 待审批列表：当前登录人作为审批人的 PENDING 记录

## 7. 首页大屏

### 7.1 今日概况
- 今日提交人数、今日总工时
- 数据范围：当天所有 PENDING/APPROVED 的工时记录

### 7.2 我的待办
- 当前登录人作为审批人的待审批条数
- 点击可跳转到审批页面

### 7.3 本月工时完成率
- 本月总工时 / (本月工作日天数 × 8 × 人数) 或自定义口径
- 按部门/项目维度展示

### 7.4 各项目工时分布
- 本月各项目工时占比（饼图/柱状图）

### 7.5 部门整体概览
- 部门总人数、总工时、人均工时

## 8. 统计报表

### 8.1 个人统计
- 按年月筛选
- 展示：总工时、各项目工时明细、各模块工时占比

### 8.2 项目统计
- 按年月筛选
- 展示：项目总工时、各模块实报 vs 预算对比

### 8.3 部门统计
- 按年月筛选
- 展示：部门总工时、人均工时、各项目占比

### 8.4 导出
- 支持 Excel 导出（Apache POI）
- 导出内容为当前筛选条件下的数据

## 9. 系统管理

### 9.1 操作日志

| 字段 | 类型 | 约束 | 说明 |
|:-----|:-----|:-----|:-----|
| id | BIGINT | PK, 自增 | |
| user_id | BIGINT | FK→tb_user, 可空 | 操作人 |
| action | VARCHAR(50) | NOT NULL | 操作类型：CREATE/UPDATE/DELETE/LOGIN/APPROVE/REJECT |
| target | VARCHAR(100) | 可空 | 操作目标描述 |

- 所有增删改操作均记录日志
- 日志列表：分页查询，支持按操作人/操作类型/日期范围筛选
