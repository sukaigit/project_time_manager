## ADR-001: JWT 认证方案

**状态：** 已接受　　**日期：** 2026-06-14

### 上下文

系统需要支持四种角色的权限隔离，且前端为原生 HTML/JS（无后端模板渲染），需要一种无状态的认证方式。

### 方案对比

| 方案 | 描述 | 优点 | 缺点 |
|:-----|:-----|:-----|:-----|
| A: JWT | JSON Web Token，服务端签发，客户端存储 | 无状态、跨域友好、前端易用 | token吊销需额外机制 |
| B: Session | 服务端存储session，cookie传递 | 可主动吊销 | 需session存储、跨域不友好 |

### 决策

选择方案 **A**：JWT

### 理由

1. 前端是独立的原生 HTML/JS，AJAX 请求携带 JWT 最简洁
2. 无状态，不需要 Redis 存储 session
3. 角色信息直接编码在 token 中，前端可解析控制菜单显隐

### 后果

- 正面：认证流程简单，前后端分离
- 负面：token 过期前无法主动吊销，但可通过设置短过期时间（如24h）缓解
