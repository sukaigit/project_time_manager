# 项目工时管理系统 - 需求规格说明书

版本：V1.0　　日期：2026-06-17

---

## 1. 引言

### 1.1 编写目的

本文档旨在全面、准确地描述项目工时管理系统的功能需求、非功能需求、业务规则和约束条件，作为系统设计、开发实现、测试验收以及用户培训的基准依据。

### 1.2 项目背景

公司研发与交付中心需要一个工时管理系统，实现工时数据的数字化管理，支撑月度、年度绩效考核的数据化评估，同时提供首页大屏和统计报表功能。

### 1.3 项目范围

- 本次覆盖的功能：认证登录、用户管理、项目管理、任务模块管理、工时填报、工时审批、首页大屏、统计报表、系统日志、角色权限
- 本次不覆盖的功能：移动端、第三方系统对接、消息通知
- 后续版本考虑的功能：移动端适配、API 开放、自动提醒

## 3. 用户角色

| 角色 | 描述 | 可访问功能 | 权限说明 |
|:-----|:-----|:-----------|:---------|
| USER | 普通用户，填写工时 | 工时填报、个人报表 | 增删改查自己的工时 |
| PM | 项目经理，维护项目+审批工时 | 项目管理、模块管理、工时审批、报表 | 管理所属项目，审批普通用户工时 |
| DEPT_MANAGER | 部门经理，全量查看 | 大屏、报表、审批（PM的工时） | 部门级数据查看，审批PM的工时 |
| ADMIN | 系统管理员 | 全部功能 | 系统最高权限 |

## 5. 非功能需求

| 类别 | 要求 | 验收标准 |
|:-----|:-----|:---------|
| 性能 | 列表接口响应时间 | P95 < 500ms |
| 安全 | 密码BCrypt加密 + JWT认证 + 验证码 | 无明文密码传输 |
| 可用性 | 系统正常工作 | 7x8小时可用 |
| 扩展性 | 支持100+用户并发 | 正常响应 |

| 术语 | 说明 |

| REJECTED | 已驳回状态 |
