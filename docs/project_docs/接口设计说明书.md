# 项目工时管理系统 - 接口设计说明书

版本：V1.0　　日期：2026-06-17

---

## 1. 接口规范

### 1.1 通用约定

| 项目 | 内容 |
|:-----|:-----|
| 基础路径 | http://localhost:8080/api |
| 协议 | HTTP/1.1 |
| 认证方式 | JWT Bearer Token |
| 数据格式 | JSON（UTF-8） |

### 1.2 统一响应格式

成功响应：
```
{"code": 200, "message": "success", "data": {}}
```

失败响应：
```
{"code": 400, "message": "错误描述", "data": null}
```

### 1.3 通用状态码

| code | 说明 |
|:-----|:-----|
| 200 | 成功 |
| 400 | 参数错误/业务校验失败 |
| 401 | 未登录/Token过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

## 2. 接口列表

### 2.1 认证接口

**POST /api/auth/login** — 登录
参数：username, password, captchaCode, captchaId
返回：{token, userId, username, name, role, firstLogin}

**GET /api/auth/captcha** — 获取验证码
返回：图片流（Content-Type: image/png）
响应头含 captcha-id

**POST /api/auth/change-password** — 修改密码
参数：oldPassword, newPassword

### 2.2 用户管理接口

| 方法 | 路径 | 说明 | 权限 |
|:-----|:-----|:-----|:-----|
| GET | /api/users | 用户列表（page, size, keyword） | ADMIN |
